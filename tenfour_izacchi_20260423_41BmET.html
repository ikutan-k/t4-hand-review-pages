  • 今回のズレは、pot odds を pre-node の大まかな range に当ててしまったことと、flat の質 や uncapped player が後ろに残る不利 を軽く見たことから起きています。

持ち帰り

  • solver の exact / 近似 node があるときは、まずそのアクションを基準線として受け入れ、なぜそうなるか を line の質から説明した方がぶれにくいです。
  • multiway preflop all-in では、pot odds を見る前に action が進んだ後の条件付きレンジ を見る必要があります。pre-node の大まかな range をそのまま当てるとズレやすいです。
  • JJ のような middle-high pair は、今どれだけ強いか だけでなく flat した後の line がどれだけ悪いか で jam が増えることがあります。
  • AKo と AKs の差は、単に数％の EQ 差ではなく、multiway all-in での実現率差として効くことがあります。

exploit 補足

  • BB の cold 4bet が solver よりさらに wide で、しかも jam に対して overfold する pool なら、BTN JJ jam は exploit でもさらに押しやすくなります。
  • 逆に cold 4bet -> call off が極端に tight な相手ばかりなら、BTN jam の利益は落ちやすく、SB AKo fold はなおさら実戦的です。
  • BB 側も、相手 2 人の all-in / call が solver より loose なら QQ はさらに降りやすく、逆に両者が underjam / undercall なら call が戻る余地があります。
